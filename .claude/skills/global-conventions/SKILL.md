## When to use this skill

- When organizing files and directories in the project structure
- When writing git commit messages or pull request descriptions
- When creating or managing feature branches
- When working with environment variables and configuration
- When updating README files or project documentation
- When managing dependencies in package.json, requirements.txt, etc.
- When establishing or following code review processes
- When using feature flags for incomplete functionality
- When maintaining changelog or release notes
- When deciding on testing requirements before merging
- When setting up or modifying project structure
- When handling secrets and API keys securely
- When contributing to the project following team guidelines
- When editing configuration files at the project root level
