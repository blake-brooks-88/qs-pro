description: Build accessible user interfaces following WCAG guidelines with semantic HTML, keyboard navigation, ARIA attributes, and screen reader support. Use this skill when creating or modifying UI components, adding interactive elements, implementing forms, building modals or dialogs, managing focus states, or ensuring color contrast compliance. Apply when editing React components, HTML templates, JSX/TSX files, or any frontend code that renders user-facing interfaces.
---

## When to use this skill

- When creating new UI components or modifying existing ones
- When adding interactive elements (buttons, links, menus, tabs)
- When implementing forms with labels and error messages
- When building modals, dialogs, or overlay components
- When managing keyboard navigation and focus states
- When adding ARIA attributes to enhance component accessibility
- When ensuring proper color contrast ratios (4.5:1 for normal text)
- When writing alternative text for images
- When structuring heading levels (h1-h6) for proper document outline
- When testing with screen readers or accessibility tools
- When editing .jsx, .tsx, .vue, .svelte, or HTML template files
- When implementing skip links or landmark regions
- When handling focus trapping in modals or dropdowns
- When making dynamic content updates accessible to assistive technologies
