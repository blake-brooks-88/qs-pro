## When to use this skill

- When writing new code in any programming language
- When naming variables, functions, classes, constants, or files
- When deciding on code formatting and indentation
- When refactoring code to improve readability
- When removing dead code, unused imports, or commented-out blocks
- When applying the DRY (Don't Repeat Yourself) principle
- When extracting common logic into reusable functions or modules
- When keeping functions small and focused on a single task
- When choosing descriptive names that reveal intent
- When editing any source code file (.ts, .tsx, .js, .jsx, .py, .go, .rs, etc.)
- When reviewing code for consistency with project conventions
- When deciding whether backward compatibility code is needed
- When maintaining consistent indentation and line breaks
