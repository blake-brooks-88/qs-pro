description: Write and organize CSS using consistent methodologies like Tailwind CSS, BEM, CSS Modules, or utility classes while maintaining design system tokens. Use this skill when styling components, writing CSS classes, using Tailwind utilities, defining design tokens (colors, spacing, typography), or optimizing CSS for production. Apply when editing .css, .scss, .less files, Tailwind config files, CSS-in-JS styled components, or any file containing style definitions.
---

## When to use this skill

- When styling new components or modifying existing styles
- When writing CSS classes, selectors, or style rules
- When using Tailwind CSS utility classes in components
- When following BEM naming conventions for CSS classes
- When working with CSS Modules or scoped styles
- When defining or using design tokens (colors, spacing, typography, shadows)
- When maintaining consistency with the project's CSS methodology
- When avoiding excessive overrides of framework styles
- When optimizing CSS for production (purging unused styles, tree-shaking)
- When editing .css, .scss, .sass, .less, or .styl files
- When working with tailwind.config.js or theme configuration files
- When using CSS-in-JS libraries (styled-components, Emotion, etc.)
- When creating responsive styles with media queries or breakpoint utilities
- When implementing dark mode or theme switching styles
